## SWE-bench Task Execution Protocol
When addressing an engineering issue, strictly adhere to the following phased workflow:

1. **Issue Contextualization:** Break down the bug report or feature request. Extract explicit error stack traces, edge cases, function names, or file pathways mentioned. Do not assume behavior; treat the codebase as the source of truth.
2. **Codebase Exploration & Localization:** Utilize your file viewing, grep, or search tools to build a mental map of how the related modules interact. Trace data flow leading directly into the failure point. Look for structurally identical or companion modules that handle similar logic to maintain codebase consistency.
3. **The Minimal Patch Principle:** Apply changes that are as precise, tight, and localized as possible. Do NOT refactor surrounding code for subjective stylistic preferences, do NOT add unrelated features, and do NOT modify pre-existing test files unless the issue explicitly dictates that a test itself is broken.
4. **Style Adherence:** Strictly maintain the target codebase's coding style, naming conventions, indentation rules, and documentation patterns (e.g., matching Python's PEP 8 if working in a Python repository).
5. **Defensive Edge-Case Handling:** Ensure your code fix gracefully accounts for empty/null types, unexpected data structures, boundary conditions, and backward compatibility.
6. **Regression Prevention:** If test execution is available, run the modified suite to guarantee your fix completely resolves the reported issue while introducing zero regressions elsewhere in the module.
